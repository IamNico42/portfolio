## TCP Congestion Control

**cwnd (Congestion Window)**
- Sender-seitiger Wert
- Gibt an wie viele Daten gleichzeitig ohne ACK gesendet werden dürfen
- ACKs -> cwnd größer
- Paketverlust -> cwnd kleiner

**IW (Initial Window)**
- Startwert von cwnd
- Beispiel: IW = 2 MSS -> 2 Segmente sofort sendbar

**ACK**
- Empfangsbestätigung
- Zeigt TCP, dass Daten angekommen sind
- Ermöglicht Wachstum von cwnd

**RTT (Round Trip Time)**
- Zeit von Anfrage bis Antwort
- Paket hin + ACK zurück

---

## Paketverlust

**3 gleiche ACKs** (**Duplicate ACKs**)
- Fehlendes Paket vermutet
- Fast Retransmit
- Paket sofort neu senden
- cwnd halbieren

**Timeout**
- Schwerer Paketverlust
- cwnd stark reduzieren (1 MSS)
- Slow Start neu beginnen

**Warum cwnd reduzieren?**
- TCP interpretiert Verlust als mögliche Überlastung
- Weniger Daten gleichzeitig senden
- Netzwerk entlasten

## HTTP

**Ohne Pipelining**
- Request -> Antwort
- Request -> Antwort
- Request -> Antwort
- Viele RTTs

Beispiel:
```
Request HTML
← HTML

Request Bild 1
← Bild 1

Request Bild 2
← Bild 2
```

**Mit Pipelining**
- Mehrere Requests direkt senden
- Antworten trotzdem in Reihenfolge
- Weniger RTTs
- Gefahr: Head-of-Line Blocking

Beispiel:

```
Request HTML
Request Bild 1
Request Bild 2

← HTML
← Bild 1
← Bild 2
```

## MTU / MSS

**MTU**
- Maximale Paketgröße inkl. Header

**MSS**
- Maximale TCP-Nutzdaten

Beispiel:

```
MTU = 1500
IP = 20
TCP = 20

MSS = 1460
```
